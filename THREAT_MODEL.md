# THREAT_MODEL

`workspace-dev` is a package-scoped, evidence-first threat model for the published npm runtime. It documents the trust boundaries, attack surfaces, mitigations, and residual risks that are implemented in this repository today. It is not a generic threat-modeling tutorial, and it is not a security claim for the broader Workspace Dev platform or for downstream apps generated by the package.

Primary companion documents:

- [`README.md`](README.md) for operator-facing runtime behavior and supported modes.
- [`ARCHITECTURE.md`](ARCHITECTURE.md) for runtime topology, artifact layout, and import governance.
- [`SECURITY.md`](SECURITY.md) for package policy, disclosure process, and security controls.
- [`COMPLIANCE.md`](COMPLIANCE.md) for enterprise evidence mapping.
- [`ZERO_TELEMETRY.md`](ZERO_TELEMETRY.md) for telemetry boundaries and verification.

## Scope, Assets, and Non-Goals

In scope for this document:

- The local HTTP runtime and inspector shell described in [`ARCHITECTURE.md`](ARCHITECTURE.md) and implemented under `src/server/`.
- The local job engine, filesystem writes, preview export, and optional GitHub PR flow implemented under `src/job-engine/`.
- Package release evidence, publish boundary, and shipped documentation enforced by `package.json`, `scripts/validate-pack.sh`, and the release workflows.

Primary assets:

- The consumer workspace and any sync destination under that workspace root.
- Runtime artifacts under `.workspace-dev`, including generated code, preview exports, `figma.json`, `design-ir.json`, and import-session event history.
- Submit-time secrets such as `figmaAccessToken`, optional `repoToken`, and optional import-session event bearer tokens.
- Generated preview content served back into the inspector.
- Release artifacts, SBOMs, provenance, and compliance evidence.

Out of scope:

- The broader Workspace Dev platform backend and any on-prem deployment around it.
- Hardening of consumer applications after code generation.
- Network perimeter controls, browser policy, or TLS termination that operators configure outside this package.

## Default Deployment Posture

The package is designed for a local-first development workflow, but several controls are defaults rather than hard invariants:

- The server binds to `127.0.0.1` by default in `src/server/constants.ts`, but host binding remains configurable in `src/server.ts`.
- Same-origin enforcement in `src/server/request-security.ts` is a browser-origin and CSRF mitigation for write routes. It is not general API authentication for every local client that can reach the runtime.
- `Strict-Transport-Security` is disabled by default and only enabled when `FIGMAPIPE_WORKSPACE_ENABLE_HSTS=true` via `src/server/constants.ts` and `src/server/http-helpers.ts`.
- Tokens are accepted at submit time and intentionally used in process memory, as documented in [`README.md`](README.md) and [`ARCHITECTURE.md`](ARCHITECTURE.md).

## Attack Surfaces

- Browser-accessible write routes: `src/server/request-security.ts` enforces JSON-only write requests, same-origin browser metadata checks, and allowed origins derived from the configured host and port.
- Preflight and route probing: `src/server/cors-preflight.e2e.test.ts` verifies protected write-route `OPTIONS` requests fail with explicit `405` responses and without permissive CORS headers.
- Path handling for UI and preview assets: `src/server/routes.ts`, `src/server/route-params.ts`, and `src/job-engine/preview.ts` normalize paths and reject traversal attempts before serving assets.
- Local sync and generated artifact writes: `src/job-engine/target-path.ts` and `src/job-engine/local-sync.ts` constrain `targetPath`, destination roots, use path-segment symlink handling, and require fresh dry-run confirmation before apply.
- Preview HTML serving and inspector bridge injection: `src/server/request-handler.ts` injects an inspect bridge into served HTML repro output and marks preview responses as embeddable.
- Import governance writes: `src/job-engine/import-session-event-store.ts` and [`ARCHITECTURE.md`](ARCHITECTURE.md) define append-only event storage, truncated notes, session-id sanitization, and persisted approval requirements.
- Figma upstream fetch and MCP resolution: `src/job-engine/figma-source.ts` applies payload-size, retry, timeout, and circuit-breaker behavior; `src/job-engine/figma-mcp-resolver.ts` requires HTTPS by default and only allows insecure loopback HTTP when `WORKSPACE_ALLOW_INSECURE_MCP=true`.
- Optional GitHub PR automation: `src/job-engine/git-pr.ts` limits support to GitHub, creates a temporary authenticated clone under `.workspace-dev/jobs/<jobId>/repo/`, and uses an askpass helper instead of interactive prompts.

## Threats, Mitigations, and Residual Risks

| Threat                                                        | Boundary                                                       | Current mitigation                                                                                                                                                                                                                                                                                                                                                                                                          | Residual risk                                                                                                                                                                                                                                    |
| ------------------------------------------------------------- | -------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Cross-origin browser abuse of write routes                    | Browser or inspector client to local runtime                   | Same-origin metadata checks, allowed-origin derivation, JSON-only write routes, explicit `405` preflight handling, DAST coverage in `src/server/dast-smoke.e2e.test.ts` and `src/server/cors-preflight.e2e.test.ts`                                                                                                                                                                                                         | This is a browser-origin control, not a general authentication layer. If the runtime is exposed beyond a trusted local deployment, non-browser clients that can reach it remain in scope for operator controls.                                  |
| Oversized or malformed request bodies                         | Browser or inspector client to local runtime                   | `src/server/http-helpers.ts` enforces deterministic JSON parsing and request-body limits, with `MAX_REQUEST_BODY_BYTES` and submit-specific larger limits in `src/server/constants.ts`                                                                                                                                                                                                                                      | Availability pressure is reduced, not eliminated. Operators still own host-level resource limits and exposure.                                                                                                                                   |
| Path traversal into UI assets, preview files, or sync targets | URL path input and runtime to workspace filesystem             | Route normalization rejects absolute paths, `..`, encoded separators, Windows paths, and NUL bytes; preview and local sync perform path-segment symlink checks; local sync revalidates with stale-preview detection before apply; best-effort `O_NOFOLLOW` is used for preview reads and sync writes when Node exposes it                                                                                                   | Exploitation still requires local filesystem write access and residual TOCTOU risk remains if an attacker swaps an ancestor path component after validation. `O_NOFOLLOW` only protects the final path component and is not portable to Windows. |
| Forged or reordered import-governance state                   | Runtime to import-session governance store                     | Bearer-only writes on both `POST /workspace/import-sessions/:id/events` and `POST /workspace/import-sessions/:id/approve`, fail-closed `503` when the bearer token is unconfigured, append-only persisted envelopes, metadata shape restrictions, `409` for impossible transitions, and repo-backed `securitySensitivePatterns` matching through `src/server/inspector-policy.ts` and `src/job-engine/import-governance.ts` | Review enforcement still depends on operators configuring the bearer token when remote or automated writers are used and on maintainers keeping the security-sensitive pattern set current for their repo.                                       |
| Secret leakage through logs or surfaced errors                | Runtime to local commands, GitHub PR flow, and public job APIs | `src/job-engine/command-runner.ts` redacts configured secrets and truncates output; `src/error-sanitization.ts` and `src/logging.ts` sanitize surfaced errors and log messages                                                                                                                                                                                                                                              | Tokens still exist in process memory during execution, and sensitive local state may exist under `.workspace-dev` while a job is active.                                                                                                         |
| Malicious or unexpected preview behavior                      | Runtime to generated preview iframe                            | Preview responses use no-store caching; preview assets remain path-contained; the inspector iframe is sandboxed in `ui-src/src/features/workspace/inspector/PreviewPane.tsx`                                                                                                                                                                                                                                                | The preview iframe deliberately runs with `allow-scripts allow-same-origin`, and preview responses drop CSP and `X-Frame-Options` to allow embedding. Treat generated preview content as untrusted local content.                                |
| Insecure or unavailable upstream Figma connectivity           | Runtime to Figma inputs and upstream services                  | Figma REST fetches apply timeout, retry, rate-limit handling, and circuit-breaker behavior; MCP resolution requires HTTPS by default and warns when loopback HTTP is explicitly allowed                                                                                                                                                                                                                                     | Availability still depends on upstream Figma and local network conditions. Operator-supplied tokens remain high-sensitivity inputs.                                                                                                              |
| Abuse or leakage in optional GitHub PR automation             | Runtime to spawned local commands and GitHub PR flow           | GitHub-only provider support, askpass helper, output redaction, bounded command capture, temp repo clone under job output, and deterministic generated project copy behavior                                                                                                                                                                                                                                                | Enabling `enableGitPr=true` expands the trust boundary and creates a temporary authenticated clone under `.workspace-dev/jobs/<jobId>/repo/`. Operators must treat the output root as sensitive local state.                                     |

## Residual Risks and Operator Assumptions

- The default HTTP bind is `127.0.0.1`, but it is configurable. Treat non-localhost binds as an explicit expansion of the trust boundary.
- Same-origin checks protect browser write flows. They should not be described as a substitute for authenticated remote APIs.
- `Strict-Transport-Security` is opt-in and environment-controlled. The runtime does not verify whether a trusted HTTPS terminator is actually present.
- Preview embedding is intentionally permissive enough to support the inspector. That makes the preview boundary weaker than the main UI shell boundary.
- Preview export and local-sync hardening assume the attacker does not already have arbitrary local filesystem write access. If they do, path-segment validation plus stale-preview detection reduce but do not eliminate ancestor-component TOCTOU races.
- Best-effort `O_NOFOLLOW` closes only the final-component symlink case on supported platforms. Windows portability is not guaranteed, so unsupported runtimes fall back to the existing fail-closed validation path.
- The package is zero telemetry by policy, but it still makes network calls that are necessary for Figma fetches, optional GitHub PR automation, and release workflows documented in [`ZERO_TELEMETRY.md`](ZERO_TELEMETRY.md) and [`COMPLIANCE.md`](COMPLIANCE.md).
- `.workspace-dev` is not just a build cache. When jobs run with secrets, previews, or Git/PR mode enabled, it contains sensitive local state that should not be treated as disposable public output.
