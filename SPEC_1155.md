# Spec: Issue #1155 - Error Chain Redaction

## Problem Statement

The redaction helpers operate on `error.message` only. They don't traverse `error.cause` chains, allowing secrets to leak when:

- `createPipelineError({ cause: fetchErr })` wraps an error with sensitive data
- A caller logs `error.stack` or uses `util.inspect(err)`
- Node 16+ renders `[cause]: Error: ...` lines containing raw tokens

## Solution

Implement a `redactErrorChain(err, seen, depthMax)` helper that:

1. Walks `error.cause` chains with cycle detection
2. Applies `redactHighRiskSecrets` to each level's message and stack
3. Returns a sanitized clone
4. Integrates into all error rendering call sites

- Redaction patterns: Bearer tokens, API keys, JSON fields all redacted
