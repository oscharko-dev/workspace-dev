# Customer-Eval Sample Plan Non-Update Note

Use this file only when a commit changes one or both customer-eval rubrics
under `fixtures/test-intelligence/customer-evals/` but does **not** require a
content change to `SAMPLE-PLAN.md`.

## How To Use

Append a short entry with:

- date;
- changed rubric file(s);
- why the sample-plan coverage and traceability matrix remain accurate;
- reviewer or issue reference for the non-update decision.

## Entry Template

```md
### 2026-05-10

- Rubric files changed: `...`
- Non-update rationale: `...`
- Reviewed in: `Issue #....` / `PR #....`
```
